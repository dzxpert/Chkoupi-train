⚡ Introduction

Chkoupi-lang is a compiled language inspired by Algerian Darija dialect. It compiles via LLVM and runs using a built-in JIT engine.

What is Chkoupi-lang?

Chkoupi-lang is a statically-typed, compiled programming language that uses Algerian dialect (Darija) keywords. Source files use the .dz extension (a nod to Algeria's country code).

The compiler uses LLVM as its backend, supporting:

  • Direct JIT execution — run code immediately
  • LLVM IR emission for inspection
  • Native object file compilation

Your First Program

Create a file salam.dz and write:

chkoupi (.dz) 
ektb("salam bzzaf!\n");

Run it with:

powershell 
chkoupi.exe salam.dz

Format Rules

  • Statements end with ;
  • Blocks use { }
  • Type annotations: dir x : tabi3i = 5;
  • Function return type: dalla f() -> tabi3i { ... }
  • Source files use the .dz extension

📦 Variables

Declare mutable variables with dir and constants with dima.

Mutable Variables — dir

Use dir to declare a variable you can reassign later.

chkoupi (.dz) 
dir x : tabi3i = 42;
dir name : nass = "brahim";
dir flag : 5iyar = sa7;
ektb("%lld\n", x);

Constants — dima

dima means "always" in Darija. A constant cannot be reassigned after declaration.

chkoupi (.dz) 
dima PI : 3ouchri = 3.14159;
ektb("PI = %lf\n", PI);

Type Inference

Type annotation is optional. The compiler infers the type from the value:

chkoupi (.dz) 
dir x = 10;    // inferred: tabi3i
dir y = 3.14;  // inferred: 3ouchri
ektb("%lld %lf\n", x, y);
💡

When the type can be unambiguously determined from the assigned value, you can omit the : type annotation entirely.

🔷 Types

Chkoupi-lang is statically typed. Every value has a type known at compile time.

Built-in Types

Keyword Type Size Example
tabi3i Integer 64-bit 42
3ouchri Float 64-bit double 3.14
5iyar Boolean — sa7 / ghalt
7arf Character 8-bit 'a'
nass String pointer "salam"
fargh Void — — (no return)

Type Examples

chkoupi (.dz) 
dir count   : tabi3i  = 100;
dir ratio   : 3ouchri = 1.618;
dir enabled : 5iyar   = sa7;
dir letter  : 7arf    = 'Z';
dir msg     : nass    = "labas!";

ektb("%lld %lf %s\n", count, ratio, msg);

Boolean Values

Booleans use the Darija words for true and false:

chkoupi (.dz) 
dir win  : 5iyar = sa7;    // true  (sa7 = correct)
dir loss : 5iyar = ghalt;  // false (ghalt = wrong)

➕ Operators

Chkoupi-lang supports standard arithmetic and comparison operators, plus Darija logical operators.

Arithmetic Operators

chkoupi (.dz) 
dir a = 10;
dir b = 3;
ektb("%lld\n", a + b);  // 13
ektb("%lld\n", a - b);  // 7
ektb("%lld\n", a * b);  // 30
ektb("%lld\n", a / b);  // 3
ektb("%lld\n", a % b);  // 1

Comparison Operators

chkoupi (.dz) 
dir x = 5;
dir y = 10;
x == y   // false — equal
x != y   // true  — not equal
x <  y   // true  — less than
x >  y   // false — greater than
x <= y   // true  — less or equal
x >= y   // false — greater or equal

Logical Operators (Darija)

Operator Meaning English Equiv.
w and (و) &&
wla or (ولا) ||
machi not (ماشي) !
chkoupi (.dz) 
dir a : 5iyar = sa7;
dir b : 5iyar = ghalt;
dir c = a w b;       // sa7 AND ghalt = ghalt
dir d = a wla b;     // sa7 OR  ghalt = sa7
dir e = machi a;    // NOT sa7 = ghalt

🔀 Control Flow

Chkoupi-lang has all the control flow structures you need, named in Darija.

If / Else — idha / idha_mknch

idha = "if", idha_mknch = "if not / else"

chkoupi (.dz) 
dir x : tabi3i = 7;
idha (x > 0) {
    ektb("positive!\n");
} idha_mknch {
    ektb("machi positive\n");
}

While Loop — ab9a_dor

ab9a_dor = "keep turning / keep looping"

chkoupi (.dz) 
dir i : tabi3i = 0;
ab9a_dor (i < 5) {
    ektb("%lld\n", i);
    i = i + 1;
}

For Loop — dor

dor = "turn / iterate"

chkoupi (.dz) 
dor (dir i = 0; i < 5; i = i + 1) {
    ektb("i = %lld\n", i);
}

Switch / Case — bdl / khyr

bdl = "change", khyr = "option/choice"

chkoupi (.dz) 
dir code : tabi3i = 2;
bdl (code) {
    khyr 1: ektb("wahd\n");
    khyr 2: ektb("zouj\n");
    khyr 3: ektb("tlata\n");
}

Try / Except — jarb / ila_ghalt

jarb = "try/test", ila_ghalt = "if wrong/error"

chkoupi (.dz) 
jarb {
    // risky code here
    ektb("kolchi mzyen!\n");
} ila_ghalt {
    ektb("wqe3 ghalta!\n");
}

🔧 Functions

Define reusable functions with dalla and return values with raja3.

Declaring a Function

dalla = "enter/go into", raja3 = "return"

chkoupi (.dz) 
dalla greet(name: nass) -> fargh {
    ektb("salam %s!\n", name);
}

greet("brahim");

Functions with Return Values

chkoupi (.dz) 
dalla add(a: tabi3i, b: tabi3i) -> tabi3i {
    raja3 a + b;
}

dir result = add(3, 7);
ektb("3 + 7 = %lld\n", result);

Recursive Functions

chkoupi (.dz) 
dalla factorial(n: tabi3i) -> tabi3i {
    idha (n <= 1) {
        raja3 1;
    }
    raja3 n * factorial(n - 1);
}

ektb("%lld\n", factorial(5));

📟 Input / Output

Use ektb to print and a9ra to read user input.

Printing — ektb

ektb means "write" in Darija. It works like C's printf with format specifiers.

Specifier Type
%lld Integer (tabi3i)
%lf Float (3ouchri)
%s String (nass)
%c Character (7arf)
chkoupi (.dz) 
ektb("salam!\n");
dir x : tabi3i = 99;
ektb("x = %lld\n", x);
dir pi : 3ouchri = 3.14;
ektb("pi = %lf\n", pi);

Reading Input — a9ra

a9ra means "read". It reads user input into a variable.

chkoupi (.dz) 
dir n : tabi3i = 0;
ektb("3tini raqm: ");
a9ra(n);
ektb("dkhalt: %lld\n", n);

💬 Comments

Use comments to document your code — they are ignored by the compiler.

Single-Line Comments

chkoupi (.dz) 
// hada comment b sahh
dir x = 42;  // inline comment

Multi-Line Comments

chkoupi (.dz) 
/*
  hadi function
  kathseb factorial
  dial ay raqm
*/
dalla factorial(n: tabi3i) -> tabi3i {
    raja3 n;
}

📥 Import

Link external Chkoupi modules into your program using jibli.

jibli — "bring me"

The jibli keyword means "bring me" in Darija. It is used to import external .dz modules.

chkoupi (.dz) 
jibli "utils";    // links utils.dz module
jibli "math";     // links math.dz module
🚧

Future feature: Module imports are planned and the syntax is finalized, but full linker integration is still under development.

🧪 Examples

Factorial Calculator

This program reads a number from the user and prints its factorial. It uses functions, recursion, I/O, and control flow.

chkoupi (.dz) 
// Factorial — chkoupi example

dalla factorial(n: tabi3i) -> tabi3i {
    idha (n <= 1) {
        raja3 1;
    }
    raja3 n * factorial(n - 1);
}

dir n : tabi3i = 0;
ektb("3tini raqm: ");
a9ra(n);
ektb("%lld! = %lld\n", n, factorial(n));

Counter Loop

chkoupi (.dz) 
// Count from 1 to 10
dor (dir i = 1; i <= 10; i = i + 1) {
    ektb("%lld\n", i);
}

FizzBuzz

Classic interview problem — print numbers 1–20, replacing multiples of 3 with Fizz, multiples of 5 with Buzz, and both with FizzBuzz.

chkoupi (.dz) 
// FizzBuzz — 1 to 20
dor (dir i = 1; i <= 20; i = i + 1) {
    idha (i % 15 == 0) {
        ektb("FizzBuzz\n");
    } idha_mknch {
        idha (i % 3 == 0) {
            ektb("Fizz\n");
        } idha_mknch {
            idha (i % 5 == 0) {
                ektb("Buzz\n");
            } idha_mknch {
                ektb("%lld\n", i);
            }
        }
    }
}

Fibonacci Sequence

Print the first 15 numbers of the Fibonacci sequence using an iterative loop.

chkoupi (.dz) 
// Fibonacci — first 15 numbers
dir a : tabi3i = 0;
dir b : tabi3i = 1;
dir tmp : tabi3i = 0;
dir count : tabi3i = 0;

ab9a_dor (count < 15) {
    ektb("%lld\n", a);
    tmp = a + b;
    a = b;
    b = tmp;
    count = count + 1;
}

⚙️ Compiler Usage

The Chkoupi compiler supports JIT execution, IR emission, and native object compilation.

JIT Execution (Default)

Run a .dz file directly — the compiler JITs and executes it immediately.

powershell 
chkoupi.exe myfile.dz

Emit LLVM IR

Print the generated LLVM Intermediate Representation for inspection.

powershell 
chkoupi.exe myfile.dz --emit-ir

Compile to Native Binary

Emit an object file, then link it with clang to produce a native executable.

powershell 
# Step 1: compile to object file
chkoupi.exe myfile.dz --emit-obj out.o

# Step 2: link with clang
clang out.o -o myprogram

# Step 3: run
./myprogram
💡

The JIT mode (default) is best for development and learning. Use --emit-obj for production deployments.
